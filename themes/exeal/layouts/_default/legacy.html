    Legacy Code Training

    No tener los conocimientos necesarios para modificar Legacy Code te está haciendo perder mucho tiempo, esfuerzo y confianza dentro del equipo de tu empresa.

    Inscríbete

    A veces, parece que la mejor solución para evitar trabajar sobre Legacy Code es "parar las máquinas" y reescribir el código desde cero. La experiencia nos dice que no.

    Enfrentarte a Legacy Code es solo cuestión de tiempo, aunque vuelvas a escribir el código desde cero, en menos tiempo del que piensas ese código que has escrito va a ser por su naturaleza irremediablemente Legacy Code.

    Trabajar sobre un proyecto de Legacy Code sin miedo a "romper" todo el sistema, es posible.

    Con las habilidades y herramientas adecuadas puedes corregir, editar y añadir nuevas funcionalidades sobre un proyecto de Legacy Code ahorrándote meses de trabajo, esfuerzo y dinero.

    Es lo que hemos aprendido con la experiencia y nuestros alumnos lo corroboran.

    El programa "Legacy Code Training" de Exeal es diferente.

    Durante 2 días (8 horas cada día) vas a trabajar teoría y práctica sobre un proyecto de Legacy Code.

    El formato del programa se basa en “pair programming” mediante ejercicios de katas, por videoconferencia en el que vas a poder consultar al tutor sobre cualquier duda o pregunta que te surja.

    El programa intensivo es personalizado y adaptable a cada persona o grupo, por ese motivo y para ofrecer el mejor servicio nuestras plazas están limitadas a 12 personas.

    Podrás trabajar sobre tu propio proyecto de Legacy Code siempre que sea open source o, en el caso de las empresas, el propio código de la misma.

    Una vez finalizado el programa de 2 días tendrás acceso al grupo privado de Slack “Exeal alumni”, para que junto al equipo y alumnos de otras ediciones puedas resolver tus dudas.

    Te daremos consejos y sugerencias, una vez acabado el programa, para que puedas enfrentarte tu solo al proyecto de la empresa sobre el que estás trabajando.

    En este programa intensivo 100% online, por videoconferencia, vas a aprender:

    • Identificar errores existentes en Legacy Code.
    • Entender cómo romper dependencias en código heredado.
    • Diferentes técnicas para la refactorización del código.
    • Añadir nuevas funcionalidades con éxito o corregir errores sobre Legacy Code.
    • Reglas que te ayudarán a escribir y mejorar el código heredado.
    • Cómo y cuando usar los diferentes test para identificar el código que debes modificar.
    • Separar módulos que están unidos y no se puede probar.
    • Aplicar con éxito estas herramientas a tu propio proyecto, independientemente del lenguaje en el que esté escrito.

    Durante 2 días (8 horas cada día) vas a trabajar teoría y práctica sobre un proyecto de Legacy Code.

    Esto es lo que vas a aprender durante la duración del programa.

    Descubre el programa del Día 1

    • - Introducción a Legacy Code y cómo hacer test automáticos para cambiar el código con seguridad.
    • - Los 4 principios del diseño simple
    • - Cómo cambiar el código Legacy paso a paso.
    • - Los pasos exactos para romper dependencias sin afectar al código.
    • - El sistema de refactorización más adecuado para mejorar el diseño y no perjudicar su funcionalidad.
    • - Consejos para refactorizar de forma segura.
    • - Identificar code smells en cualquier código.
    • - Los diferentes tipos de Test para modificar el código con éxito y cuándo utilizar cada uno de ellos
    • - Cómo enfrentarte a una base de código que no conoces sin morir en el intento.
    • - Diferentes herramientas y habilidades para modificar, editar y extraer el código sin perjudicar al sistema.

    Descubre el programa del Día 2

    • - Cómo separar módulos que están unidos y no se pueden probar.
    • - Cómo construir una red de seguridad para cambiar el código.
    • - Cómo arreglar bugs usando TDD.
    • - Cómo añadir una nueva funcionalidad usando TDD.
    • - Técnicas avanzadas para modificar el código y orientarte en nuevos proyectos.
    • - Estrategias para escribir sobre Legacy minimizando el riesgo y alternativas para evitar reescribirlo de nuevo.
    • - Cómo identificar de manera exacta los módulos cuya mejora supondrá un alto ROI.
